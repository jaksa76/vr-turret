# Plan: Turret Tracking Speed [Issue #1]

## Summary

The `turret` component currently snaps to the target direction instantly each tick. The fix adds `yawSpeed` and `pitchSpeed` schema properties (degrees/second). Each tick, the component computes the desired rotation, then clamps the per-frame angular step so the turret converges toward the target without exceeding the configured speeds.

## Files to Change

| File | Change |
|---|---|
| `components/turret.js` | Add schema properties + tracking-speed logic in `tick` |

No HTML changes are required; the new properties have sensible defaults and existing `turret` usages in `index.html` continue to work unchanged.

## Implementation Steps

1. **Add schema properties**
   Add `yawSpeed` and `pitchSpeed` to the component schema (type `number`, default e.g. `90` deg/s each). Having two separate speeds is realistic â€” real turrets often slew faster horizontally than they elevate.

2. **Persist current angles in `init`**
   Initialise `this.currentYaw` and `this.currentPitch` from the entity's current rotation (or `0`) so there is no discontinuity on the first tick.

3. **Compute target angles (same as today)**
   Keep the existing vector-direction â†’ `Math.asin`/`Math.atan2` logic to get `targetPitch` and `targetYaw` in degrees.

4. **Step current angles toward target, clamped by speed**
   ```
   maxYawStep   = data.yawSpeed   * (timeDelta / 1000)   // timeDelta is ms
   maxPitchStep = data.pitchSpeed * (timeDelta / 1000)

   yawDelta   = shortestAngleDelta(currentYaw,   targetYaw)
   pitchDelta = shortestAngleDelta(currentPitch, targetPitch)

   currentYaw   += clamp(yawDelta,   -maxYawStep,   maxYawStep)
   currentPitch += clamp(pitchDelta, -maxPitchStep, maxPitchStep)
   ```
   `shortestAngleDelta` wraps the difference into `(-180, 180]` to ensure the turret always takes the short path (avoids spinning 340Â° instead of âˆ’20Â°).

5. **Apply the stepped rotation**
   Replace `this.el.setAttribute('rotation', rotation)` with the stepped values.

6. **Hold position when there is no intersection**
   When `intersections.length === 0`, do nothing â€” the turret stays at its current angles rather than snapping to a default.

## Risks / Edge Cases

| Risk | Mitigation |
|---|---|
| **Angle wrap-around** | Use a `shortestAngleDelta` helper that wraps the delta into `(-180, 180]` before clamping. |
| **`timeDelta` spike on first tick or tab-switch** | A-Frame can deliver very large `timeDelta` on the first frame (~1 s+). Cap `timeDelta` at e.g. 100 ms before multiplying to avoid a single huge step that overshoots the target. |
| **`currentYaw`/`currentPitch` drift** | After clamping, the stored angles can drift from the entity's actual rendered rotation if something else also writes rotation. Keep the stored state as the single source of truth and always apply it via `setAttribute`. |
| **Pitch limits** | Real turrets have physical elevation limits (e.g. âˆ’10Â° to +85Â°). This issue does not ask for limits, but the implementation should make it easy to add `minPitch`/`maxPitch` later. |
| **`rotation-mimic` component on the turret mid section** | That component copies Y rotation from `#left-cannon` / `#right-cannon` each tick. It reads `object3D.rotation.y` which will now lag correctly â€” no change needed there. |
