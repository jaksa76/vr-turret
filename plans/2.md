# Plan: Turret Tracking Sound

## Summary

Add a looping servo sound to the `turret` component that plays while the turret is actively rotating and adjusts playback rate (pitch) proportional to the turret's angular speed. The sound asset path, minimum playback rate, and maximum playback rate are all configurable via component schema properties.

## Files to Change

- `components/turret.js` — add schema properties; track previous rotation to compute angular speed; drive a looping sound with dynamic playback rate each tick.
- `index.html` — declare a servo audio asset in `<a-assets>` and add the new properties to both `<a-entity turret>` elements.
- `assets/` — a `servo.wav` (or similar) file is expected here; a placeholder must be present so the attribute is wired up, but the actual sample will be provided externally.

## Implementation Steps

1. **Extend `turret.js` schema** — add three new properties:
   ```js
   servoSound:      { type: 'string', default: '' },   // asset selector or URL; empty = no sound
   minPlaybackRate: { type: 'number', default: 0.5 },  // rate when moving slowly
   maxPlaybackRate: { type: 'number', default: 2.0 },  // rate at maxAngularSpeed
   maxAngularSpeed: { type: 'number', default: 180 }   // deg/sec that maps to maxPlaybackRate
   ```

2. **`init` — set up state tracking** — add instance fields:
   ```js
   this.prevRotation = new THREE.Euler();   // scratch: previous rotation (degrees)
   this.servoReady   = false;               // audio buffer is loaded
   this.servoPlaying = false;               // loop is currently running
   ```
   If `data.servoSound` is non-empty, programmatically add a looping sound component to the element:
   ```js
   this.el.setAttribute('sound', `src: ${this.data.servoSound}; loop: true; autoplay: false; poolSize: 1`);
   this.el.addEventListener('sound-loaded', () => { this.servoReady = true; });
   ```

3. **`tick` — compute angular speed** — after the rotation is updated via `setAttribute`:
   - Read the new rotation from `this.el.object3D.rotation` (in radians), convert to degrees.
   - Compute delta: `Δpitch = |newX - prevX|`, `Δyaw = |newY - prevY|` (in degrees, handle wrap-around with `% 360` clamping).
   - `angularSpeed = (Δpitch + Δyaw) / (timeDelta / 1000)` in deg/sec.
   - Store current rotation in `this.prevRotation` for the next frame.

4. **`tick` — drive sound** — using the computed angular speed:
   - If `angularSpeed > deadzone` (e.g. 0.5 deg/sec) and `servoReady`:
     - If not yet playing: call `this.el.components.sound.playSound()`, set `this.servoPlaying = true`.
     - Map speed to playback rate:
       ```js
       const t = THREE.MathUtils.clamp(angularSpeed / this.data.maxAngularSpeed, 0, 1);
       const rate = THREE.MathUtils.lerp(this.data.minPlaybackRate, this.data.maxPlaybackRate, t);
       ```
     - Apply to the Web Audio source node (valid while playing):
       ```js
       const sound = this.el.components.sound.sound;
       if (sound && sound.source) sound.source.playbackRate.value = rate;
       ```
   - If `angularSpeed <= deadzone` and `servoPlaying`:
     - Call `this.el.components.sound.stopSound()`, set `this.servoPlaying = false`.

5. **`remove`** — stop the sound if playing, to prevent orphaned audio when the entity is removed.

6. **`index.html` asset** — add inside `<a-assets>`:
   ```html
   <audio id="servoSound" src="assets/servo.wav" preload="auto"></audio>
   ```

7. **`index.html` turret entities** — add the new attributes to both `<a-entity turret>` elements:
   ```html
   turret="target: #cursor; servoSound: #servoSound; minPlaybackRate: 0.5; maxPlaybackRate: 2.0; maxAngularSpeed: 180"
   ```

## Testing

- **Desktop — idle:** Open `index.html`; no servo sound should play when the cursor is not moving.
- **Desktop — movement:** Move the mouse over the scene; servo sound should start playing and the pitch should rise with faster cursor movement and drop with slow movement.
- **Pitch scaling:** Sweep the cursor from one side of the screen to the other quickly; verify the rate audibly reaches the high end. Slow, steady movement should produce a lower pitch near `minPlaybackRate`.
- **Stop on idle:** Hold the mouse still for a moment; the servo loop should stop.
- **Configurable properties:** Temporarily change `minPlaybackRate: 1.0; maxPlaybackRate: 1.0` in the HTML — pitch should remain constant regardless of speed. Change `maxAngularSpeed` to a small value and verify the max pitch is reached quickly.
- **Both turrets:** Ensure each turret emits sound independently (they are separate entities with separate sound components).
- **VR headset:** Verify the sound is audible and spatially positioned near each turret on the Meta Quest.
- **Entity removal:** If a turret is removed mid-game, confirm no audio artifacts remain.

## Risks / Edge Cases

- **`sound.source` lifetime:** `AudioBufferSourceNode` is single-use — `source` is recreated on each `playSound()` call. Only access `sound.source.playbackRate.value` after confirming the sound is playing (`servoPlaying === true`). Add a null guard before writing the property.
- **`sound-loaded` vs `update` ordering:** If `update` is called (e.g. target switch) before the sound loads, the `servoReady` flag will still be false. This is safe — the sound simply won't play until it is ready.
- **Wrap-around delta:** `atan2` output in `turret.js` can jump across the ±180° boundary on a single tick. Clamp `Math.abs(delta) % 360` and take `Math.min(delta, 360 - delta)` to avoid false high-speed spikes.
- **A-Frame `sound` component conflict:** If the turret entity already has a `sound` attribute (it doesn't currently, but future refactors might), `setAttribute('sound', ...)` in `init` will overwrite it. Use a dedicated component attribute name or check first.
- **timeDelta = 0 on first tick:** Guard against division by zero when computing deg/sec: `if (timeDelta === 0) return;`.
- **VR latency:** On the Quest, the audio context may start suspended. A-Frame handles resuming the context on user gesture, but the first frame of movement may produce no sound. This is acceptable browser behavior.
- **Multiple turrets, one audio context:** Both turrets share the browser's single AudioContext. Using `poolSize: 1` per turret is correct; do not use a shared global audio node.
