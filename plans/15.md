# Plan: Issue 15 — Plasma Weapon

## Summary

Add a plasma weapon as a secondary fire mode alongside the existing bullet shooter. Plasma bolts are large, slow-moving, glowing spheres that deal more damage per hit (25 vs 10 for bullets) but fire more slowly and travel at lower speed. The player fires plasma via the Y button on the left Oculus Touch controller, or the `P` key in mouse mode.

Plasma is visually distinct from bullets: a larger cyan/blue emissive sphere with an attached point light for a glow effect. This makes it clear to the player that a different weapon is active.

## Implementation Steps

### 1. Create `components/plasma-projectile.js`

Register an A-Frame component `plasma-projectile`:

- **Schema**: `{ velocity: { type: 'vec3', default: {x:0, y:0, z:-0.01} } }`
- **init**:
  - Set sphere geometry (radius `0.18`) with emissive cyan material (`color: #0CF`, emissive: `#0CF`, emissiveIntensity: `1.5`).
  - Append a child `<a-entity light="type:point; color:#0CF; intensity:1.5; distance:4">` for glow.
  - The entity is created with `ammo-body="type: kinematic"`, `ammo-shape="type: sphere"`, and `class="plasma"`.
- **tick(time, timeDelta)**:
  - Move position by `velocity * timeDelta` (same pattern as `projectile.js`).
  - Remove self when `position.z < -500`.
- **collidestart event**: Remove self from scene (add `hideCollider` guard like `projectile.js`).

Geometry and physics are set by `plasma-shooter` at creation time (not in `init`) to follow the same pattern as `shooter.js`.

### 2. Create `components/plasma-shooter.js`

Register an A-Frame component `plasma-shooter`:

- **Schema**: `{ cooldown: { type: 'number', default: 1200 } }`
- **State**: `this.lastShot = 0`
- **`fire(time)` method** (called externally by `control-switcher`):
  - If `time - this.lastShot < this.data.cooldown`, return early.
  - Record `this.lastShot = time`.
  - Get world position and quaternion from `this.el.object3D` (same as `shooter.js`).
  - Build velocity vector `(0, 0, -0.025)` (half the bullet speed), rotate by quaternion.
  - Create `<a-entity>`, set attributes:
    - `plasma-projectile="velocity: ..."`
    - `position`
    - `geometry="primitive: sphere; radius: 0.18"`
    - `material="color: #0CF; emissive: #0CF; emissiveIntensity: 1.5"`
    - `ammo-body="type: kinematic"`
    - `ammo-shape="type: sphere"`
    - `class="plasma"`
  - Append to scene.
  - Play sound if `this.el.components.sound` exists.

The component has no `tick` — it is purely demand-driven.

### 3. Update `components/control-switcher.js`

Currently handles `abuttondown` to toggle mouse/dual mode.

Add:
- **`ybuttondown` handler** on the right-hand controller element:
  - Call `plasmaShooterLeft.fire(performance.now())` and `plasmaShooterRight.fire(performance.now())`.
  - In dual mode, fire only the turret controlled by the right hand (or both — simpler to fire both).
- **Keyboard handler** (`document.addEventListener('keydown')`): on `KeyP`, call `fire()` on both plasma shooters.

Store references to plasma-shooter components on init by querying the shooter entities (same pattern used for turret/controller references in existing `_applyMode()`).

### 4. Update `components/destroyable.js`

Change the `hit()` method from a hard-coded bullet check to a damage lookup:

```js
hit: function(evt) {
    const cls = evt.detail.targetEl.className;
    let damage = 0;
    if (cls === 'bullet') damage = 10;
    else if (cls === 'plasma') damage = 25;
    else return;

    this.data.health -= damage;
    if (this.data.health <= 0) this.die();
}
```

This is backward-compatible: `bullet` behaviour is unchanged.

### 5. Register components and add entities in `index.html`

- Add before the closing `</head>`:
  ```html
  <script src="components/plasma-projectile.js"></script>
  <script src="components/plasma-shooter.js"></script>
  ```
- On each turret's shooter entity (there are two — left and right cannon), add the `plasma-shooter` component:
  ```html
  <a-entity id="left-shooter" shooter="" plasma-shooter="" sound="..." position="..."></a-entity>
  <a-entity id="right-shooter" shooter="" plasma-shooter="" sound="..." position="..."></a-entity>
  ```
  Adding IDs makes them easy to query in `control-switcher`.

## Testing

No automated tests exist; verification is manual (on desktop + Meta Quest):

1. **Visual check**: On load, no plasma bolts visible. Pressing `P` fires a large cyan glowing orb from the turret barrel.
2. **Fire rate**: Rapid `P` key presses only produce one bolt per ~1.2 s.
3. **Velocity**: Plasma bolt travels noticeably slower than bullets.
4. **Damage**: A standard enemy (20 HP) dies in one plasma hit; a 30 HP enemy requires two hits.
5. **Cleanup**: Plasma bolts that miss all enemies disappear when they travel far enough; no lingering entities in the DOM.
6. **Collision cleanup**: Hitting an enemy removes the bolt immediately; no console errors about missing `hideCollider`.
7. **Glow effect**: The point light child is visible on nearby surfaces.
8. **VR Y button**: Pressing Y on the Oculus Touch controller fires plasma from both turrets.
9. **Dual mode**: Plasma fires correctly alongside bullets when turrets are in dual-controller mode.
10. **Bullet unchanged**: Existing rapid-fire bullets continue to work and still deal 10 damage.

## Risks / Edge Cases

- **`className` exact-match fragility**: `destroyable.js` currently uses `=== 'bullet'` (exact string). The plasma entity must have `class` set to exactly `'plasma'` with no other classes. If other components ever add classes to these entities, switch to `classList.contains()` in `destroyable.js`.
- **Performance — point light**: Dynamic point lights are expensive on mobile VR (Quest). If frame rate drops, remove the child light entity and rely solely on the emissive material colour for the glow effect.
- **Physics sphere vs box**: The existing bullets use `ammo-shape="type: box"`. Plasma uses `type: sphere`. Confirm `aframe-physics-system` (ammo driver) handles sphere shapes correctly for kinematic bodies and emits `collidestart` reliably — test against an enemy with a box shape.
- **Y button event name**: On `oculus-touch-controls`, the Y button is on the left controller and emits `ybuttondown`. The right controller's equivalent button is B (`bbuttondown`). Confirm which hand should trigger plasma and use the correct event. Current plan uses right-hand Y (check A-Frame oculus-touch-controls docs for exact event names per hand).
- **`performance.now()` vs A-Frame `time`**: `control-switcher` runs outside A-Frame's `tick` loop, so `performance.now()` is used for the cooldown timestamp. `plasma-shooter.js` stores `this.lastShot` using this value. Be consistent — don't mix A-Frame tick `time` with `performance.now()`.
- **Sound asset**: No dedicated plasma sound asset currently exists. Either reuse the laser sound at a lower pitch (A-Frame `sound` component doesn't support pitch), or omit sound for now and note it as a follow-up. Plan 16 (homing missiles) faces the same issue.
- **Interaction with Plan 16 (missiles)**: Both plans add Y/X button handlers in `control-switcher`. If both are implemented, ensure the button assignments don't conflict (missiles on X, plasma on Y is a clean split).
