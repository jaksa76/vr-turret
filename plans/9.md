# Plan: Issue 9 — Energy Shield

## Summary

Add a player energy shield (100 HP) that drains when enemy projectiles hit the player. A visible HUD bar tracks the current shield level. When the shield reaches zero, an explosion spawns at the player's position and the game reloads after a short delay.

Enemy projectiles from `shooting-enemy.js` currently have no collision detection against the player — they use `projectile-motion` which simply moves the entity through space and expires after a lifetime. The simplest fix is a proximity check in `projectile-motion`'s `tick`: if the projectile comes within a threshold distance of `#camera`, it counts as a hit.

A new `player-shield` component holds shield state and handles damage, HUD updates, and the death sequence.

## Files to Change

| File | Change |
|------|--------|
| `components/player-shield.js` | **New.** A-Frame component: tracks shield HP, listens for `shield-hit` scene events, updates HUD bar, triggers death explosion + reload |

## Implementation Steps

### 1. Add proximity hit detection — `shooting-enemy.js`

In the `projectile-motion` `tick` function, after computing the new position and before any early-return, check the distance to the player camera:

```js
tick: function(time, timeDelta) {
    if (!this.el.parentNode) return;

    if (time - this.startTime > this.data.lifetime) {
        this.el.parentNode.removeChild(this.el);
        return;
    }

    const dt = timeDelta / 1000;
    this.el.object3D.position.addScaledVector(this.currentVelocity, dt);

    // Proximity hit check against player
    if (!this.camera) {
        this.camera = document.querySelector('#camera');
    }
    if (this.camera) {
        const playerPos = new THREE.Vector3();
        this.camera.object3D.getWorldPosition(playerPos);
        const projPos = new THREE.Vector3();
        this.el.object3D.getWorldPosition(projPos);
        if (projPos.distanceTo(playerPos) < 1.2) {
            this.el.sceneEl.emit('shield-hit', { damage: 20 });
            if (this.el.parentNode) this.el.parentNode.removeChild(this.el);
        }
    }
},
```

Cache `this.camera` after first resolution to avoid a `querySelector` call every frame.

The hit radius of `1.2` units and damage value of `20` are tunable — at 20 damage the shield survives 5 direct hits, giving the player meaningful gameplay time.

### 2. Create `components/player-shield.js`

```js
AFRAME.registerComponent('player-shield', {
  schema: {
    max:   { type: 'number', default: 100 },
    value: { type: 'number', default: 100 }
  },

  init() {
    this.bar   = document.getElementById('shield-bar');
    this.label = document.getElementById('shield-label');
    this.dead  = false;

    this.el.sceneEl.addEventListener('shield-hit', (evt) => {
      if (this.dead) return;
      this.data.value = Math.max(0, this.data.value - (evt.detail.damage || 20));
      this.updateHUD();
      if (this.data.value <= 0) this.die();
    });

    this.updateHUD();
  },

  updateHUD() {
    const pct = this.data.value / this.data.max;
    if (this.bar) {
      this.bar.style.width = (pct * 100) + '%';
      // Color: green → yellow → red
      if (pct > 0.5)       this.bar.style.background = '#0f0';
      else if (pct > 0.25) this.bar.style.background = '#ff0';
      else                 this.bar.style.background = '#f00';
    }
    if (this.label) {
      this.label.textContent = 'SHIELD: ' + Math.round(this.data.value);
    }
  },

  die() {
    this.dead = true;

    // Spawn explosion at camera world position
    const pos = new THREE.Vector3();
    this.el.object3D.getWorldPosition(pos);
    const explosion = document.createElement('a-entity');
    explosion.setAttribute('explosion', 'radius: 3; numberOfBlobs: 80');
    explosion.setAttribute('position', `${pos.x} ${pos.y} ${pos.z}`);
    this.el.sceneEl.appendChild(explosion);

    // Reload after delay so the explosion is visible
    setTimeout(() => location.reload(), 2000);
  }
});
```

### 3. Add HUD overlay and script tag — `index.html`

In `<head>`, after the other component scripts:
```html
<script src="components/player-shield.js"></script>
```

In `<body>`, immediately before `<a-scene>`, add the shield HUD:
```html

  bottom: 20px;

    <div id="shield-bar" style="height:100%; width:100%; background:#0f0; transition: width 0.15s;"></div>
  </div>

```

Attach `player-shield` to the camera entity:
```html
<a-camera id="camera" player-shield>
```

## Testing

| Scenario | Expected |
|----------|----------|

| Enemy projectile passes at distance > 1.2 | No damage; projectile continues/expires normally |

| Shield at ~20% | Bar turns red |

| Player never gets hit | Shield stays at 100; no interference with normal gameplay |

## Risks / Edge Cases

- **`projectile-motion` `querySelector` cost.** Caching `this.camera` on the component instance avoids repeated DOM queries. First-frame lookup on any projectile is acceptable.

- **Hit radius tuning.** A radius of 1.2 units is approximate — the player camera has no physical size. If enemies fire too frequently and hits feel unavoidable, raise the radius or reduce fire rate. Document the constant with a comment.

- **`THREE.Vector3` allocation in tick.** Creating `new THREE.Vector3()` every frame per projectile creates GC pressure. Allocate scratch vectors in `init` and reuse via `.set()` in `tick`. Flag this as a performance optimization — with a small number of simultaneous projectiles it is acceptable for now.

- **Interaction with plan 10/11 (game-state + overlay).** Plans 10 and 11 introduce a `game-state` component that shows a game-over overlay for `waves-complete`. This plan uses `location.reload()` for player death, which bypasses any overlay. When both are implemented together, the `die()` method should emit a `player-died` scene event and `game-state` should listen for it and show the overlay before reloading — rather than calling `location.reload()` directly in `player-shield`. Coordinate carefully.

- **VR HUD visibility.** A fixed-position HTML `<div>` is not rendered inside the WebXR compositor; it appears only in desktop/2D mode. For VR headset users, a 3D A-Frame `<a-text>` entity attached to `#camera` (as a child) would be needed. This is deferred — the HTML HUD is sufficient for desktop testing and the desktop is the primary development target.

- **Explosion at camera origin.** The `explosion` component uses `a-entity` appended to the scene. Since the explosion is positioned at the camera's world position (which is `0 0 0` if the camera hasn't moved), verify the world position lookup is correct via `getWorldPosition` rather than `object3D.position` (which may be local).

- **`dead` flag resets on reload.** Since `location.reload()` resets all JS state, there is no risk of the `dead` flag persisting across game sessions.

- **Interaction with plan 12 (powerup) proximity check.** Plan 12's powerup also checks proximity to `#camera` in its `tick`. The two checks are independent and do not interfere with each other.
