# Plan: Issue 16 — Secondary Weapon: Homing Missiles

## Summary

Add homing missiles as a secondary weapon alongside the existing bullet shooter. Missiles track the nearest enemy, deal higher damage, and have limited ammo displayed on a HUD. The player switches to missiles via the X button (left Oculus Touch controller) or a keyboard key in mouse mode.

---

## Files to Change

| File | Change |
|------|--------|
| `components/homing-projectile.js` | **New.** Missile entity that steers toward a target each tick with a max turn rate. Self-destructs on collision or timeout. |
| `components/missile-launcher.js` | **New.** Secondary shooter: tracks ammo count, fires homing missiles, exposes current ammo for HUD. |
| `components/control-switcher.js` | Add X-button handler to toggle active weapon between bullets and missiles. |
| `index.html` | Add `<script>` tags for new components; add `missile-launcher` to each turret cannon; add ammo HUD overlay; wire up keyboard shortcut for mouse mode. |

`shooter.js`, `projectile.js`, `destroyable.js` — no changes needed.

---

## Implementation Steps

### 1. Create `components/homing-projectile.js`

Implement an A-Frame component that:

- **Schema**: `{ speed: 30, turnRate: 2.5, lifetime: 5000, damage: 30, target: { type: 'selector' } }`
- **init**: Store velocity from `data.velocity` (set by launcher at creation), start lifetime timer that removes the entity.
- **tick(t, dt)**:
  - If `data.target` exists and is still in the scene, compute the direction from missile to target's world position.
  - Slerp/lerp current velocity direction toward that direction by `turnRate * dt / 1000`.
  - Advance position by `velocity * speed * dt / 1000`.
- **Collision**: Listen for `collidestart`; if `evt.detail.targetEl.classList.contains('bullet')` — no, missiles collide with enemies. Enemies' `destroyable` component already listens for `collidestart` and checks `evt.detail.targetEl.className == 'bullet'`. Update `destroyable.js` to also accept class `missile` for higher damage (see step 4).
- Entity is created with `ammo-body="type: kinematic"`, `ammo-shape="type: sphere"`, and class `missile`.

### 2. Create `components/missile-launcher.js`

A-Frame component attached alongside `shooter`:

- **Schema**: `{ maxAmmo: 10, cooldown: 1500, active: false }`
- **State**: `ammoCount = maxAmmo`, `lastFired = 0`, `active = false`
- **Methods**:
  - `fire(turretEl)`: If not active, ammo > 0, and cooldown elapsed → find nearest enemy (query `document.querySelectorAll('.enemy')`), compute closest by world-position distance → create missile entity pointed at it → decrement ammo → emit `ammo-changed` event on self with `{ ammo, maxAmmo }`.
  - `setActive(val)`: Toggle `active` flag; emit `weapon-changed` with active state.
- The turret's `shooter` component fires bullets each 500 ms via its own tick. `missile-launcher` does **not** auto-fire; it is triggered on demand from `control-switcher` (B/X button or keyboard).

### 3. Update `components/control-switcher.js`

Currently handles A button → toggle mouse/dual mode.

Add:
- Listen for **X button down** (`xbuttondown` on right-hand or left-hand controller — left controller's X maps to `xbuttondown` on `oculus-touch-controls`).
- On X press: call `missileLauncher.fire(turretEl)` on both turrets (or whichever is active).
- In mouse mode: also listen for keyboard event `KeyM` (or `Space`) to fire a missile.

> The A button dual-mode toggle stays unchanged.

### 4. Update `components/destroyable.js`

Change the bullet-check to handle missiles:

```js
// Before
if (evt.detail.targetEl.className == 'bullet') { damage = 10; }

// After
const cls = evt.detail.targetEl.className;
if (cls === 'bullet') damage = 10;
else if (cls === 'missile') damage = 30;
else return;
```

### 5. Add HUD to `index.html`

Add a fixed 2D HTML overlay (outside `<a-scene>`) for ammo display:

```html
<div id="hud" style="position:fixed; bottom:20px; right:20px; color:#0f0; font-family:monospace; font-size:24px; z-index:999; pointer-events:none">
  <span id="missile-ammo">🚀 10 / 10</span>
</div>
```

Wire it up via a small inline script that listens for `ammo-changed` on the missile-launcher entities and updates the text.

### 6. Register components and add entities in `index.html`

- Add `<script src="components/homing-projectile.js">` and `<script src="components/missile-launcher.js">`.
- Add `missile-launcher` component to the `<a-entity shooter ...>` inside each turret cannon:
  ```html
  <a-entity shooter="" missile-launcher="" sound="..."></a-entity>
  ```

## Testing

Since there are no automated tests in this repo, verification is manual:

1. **Ammo display**: HUD shows "10 / 10" on load.
2. **Fire missile**: Press X (controller) or M (keyboard) → one missile spawns, HUD updates to "9 / 10".
3. **Homing**: Missile visually curves toward the nearest enemy and hits it.
4. **Damage**: Enemy takes 30 damage per missile hit (vs 10 for bullets); confirmed by needing fewer missiles to destroy an enemy.
5. **Ammo exhaustion**: After 10 fires, HUD shows "0 / 10" and additional X presses do nothing.
6. **No target**: If no enemies are present, missile fires in the direction the turret is aimed (fallback: use current aim direction as velocity with no homing).
7. **Cooldown**: Rapid X presses only fire one missile per 1.5 s.
8. **Dual-mode**: Missiles fire correctly when turrets are in dual-controller mode.
9. **Missile expiry**: If missile doesn't hit anything within 5 s, it removes itself without error.

## Risks / Edge Cases

- **Enemy dies mid-flight**: The missile's target entity is removed. Guard with `data.target && data.target.parentNode` check each tick before steering.
- **Physics body creation cost**: Creating kinematic ammo bodies is more expensive than simple entities. Limit missiles to one at a time or pool them if performance degrades on Quest.
- **`className` vs `classList`**: A-Frame entities use both. The existing `destroyable.js` checks `className == 'bullet'` (exact string). The missile entity must have class set as a single class (`el.setAttribute('class', 'missile')`) or the check will fail. Alternatively refactor to `classList.contains()`.
- **X button event name**: On `oculus-touch-controls`, left controller X button emits `xbuttondown`; confirm via A-Frame docs — may differ between left/right hand or between controller models. Test on device.
- **HUD visibility in VR**: A fixed HTML overlay works in non-VR mode. In VR (Quest) the HUD won't be visible inside the headset. A follow-camera A-Frame text entity (`<a-text>`) attached to the camera rig is the correct VR solution; consider implementing both.
- **Ammo reset**: Decide whether ammo resets on page reload only or also on wave completion. No wave-end hook currently exists.
